---
title: "Analysis endpoints"
description: "Generate and retrieve AI-powered movie analyses"
---

Analysis endpoints drive the core feature of Magic Film: generating and serving structured cinematographic analyses powered by Google Gemini. Results are cached in the database — once an analysis exists, it is served instantly without a new Gemini call.

---

## POST /api/analisis/generar

Generate an AI analysis for a movie. Requires authentication.

If a complete analysis already exists for the requested movie, it is returned immediately from the database cache. If not, a new analysis is generated by calling Google Gemini 1.5 Flash.

### Request headers

<ParamField header="Authorization" type="string" required>
  Bearer token from login or registration. Example: `Bearer eyJhbGci...`
</ParamField>

### Request body

<ParamField body="pelicula_id" type="integer" required>
  Internal ID of the movie to analyse. Obtain this from the movie search or detail endpoints.
</ParamField>

### Response

<ResponseField name="success" type="boolean" required>
  `true` when analysis is returned successfully.
</ResponseField>

<ResponseField name="cached" type="boolean" required>
  `true` if the analysis was served from the database cache. `false` if it was freshly generated.
</ResponseField>

<ResponseField name="data" type="object" required>
  <Expandable title="properties" defaultOpen>
    <ResponseField name="analisis" type="object">
      The analysis record.
      <Expandable title="analisis properties">
        <ResponseField name="id" type="number">
          Analysis record ID.
        </ResponseField>
        <ResponseField name="tipo" type="string">
          Analysis type: `entretenimiento` or `profundo`.
        </ResponseField>
        <ResponseField name="estado" type="string">
          Current state: `pendiente`, `generando`, `completo`, or `error`.
        </ResponseField>
        <ResponseField name="generado_en" type="string">
          ISO 8601 timestamp of when the analysis was generated.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="capas" type="object[]">
      Array of analysis layers. Each layer contains a focused section of the analysis.
      <Expandable title="capa object">
        <ResponseField name="nombre_capa" type="string">
          Layer identifier. Entertainment mode: `resumen`, `momentos`, `easter_eggs`, `curiosidades`. Deep mode: `narrativa`, `simbolismo`, `contexto`, `tecnica`, `conclusion`.
        </ResponseField>
        <ResponseField name="contenido" type="string">
          Rich text content for this layer, generated by Gemini.
        </ResponseField>
        <ResponseField name="orden" type="number">
          Display order for this layer.
        </ResponseField>
        <ResponseField name="modo" type="string">
          Layer mode: `entretenimiento` or `profundo`.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  Generating a new analysis calls Google Gemini 1.5 Flash and typically takes **5–15 seconds**. Cached analyses are returned in under 100ms.
</Note>

### Analysis states

The `estado` field in the `analisis` record transitions through the following states:

```
pendiente → generando → completo
                     ↘ error
```

If the analysis returns with `estado: error`, the generation failed and can be retried by an admin using the [regenerate endpoint](/api/admin#put-apiadminanalisisidregenerar).

### Errors

| Status | Description |
|--------|-------------|
| `400` | `pelicula_id` is missing from the request body. |
| `401` | No valid bearer token provided. |
| `404` | No movie found with the given `pelicula_id`. |
| `500` | Gemini API call failed. |

### Example

```bash
curl --request POST \
  --url http://localhost:3000/api/analisis/generar \
  --header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...' \
  --header 'Content-Type: application/json' \
  --data '{"pelicula_id": 1}'
```

## GET /api/analisis/:peliculaId

Retrieve an existing analysis for a movie. No authentication required.

### Path parameters

<ParamField path="peliculaId" type="integer" required>
  Internal ID of the movie whose analysis you want to retrieve.
</ParamField>

### Response

<ResponseField name="success" type="boolean" required>
  `true` when an analysis is found.
</ResponseField>

<ResponseField name="data" type="object" required>
  <Expandable title="properties" defaultOpen>
    <ResponseField name="id" type="number">
      Analysis record ID.
    </ResponseField>
    <ResponseField name="tipo" type="string">
      Analysis type: `entretenimiento` or `profundo`.
    </ResponseField>
    <ResponseField name="estado" type="string">
      Current state: `pendiente`, `generando`, `completo`, or `error`.
    </ResponseField>
    <ResponseField name="generado_en" type="string">
      ISO 8601 timestamp of generation.
    </ResponseField>
    <ResponseField name="capas" type="object[]">
      Array of analysis layer objects. See [POST /api/analisis/generar](#post-apianalisisgenerar) for the full layer schema.
    </ResponseField>
  </Expandable>
</ResponseField>

### Errors

| Status | Description |
|--------|-------------|
| `404` | No analysis exists for this movie yet. Use `POST /api/analisis/generar` to create one. |

### Example

```bash
curl --request GET \
  --url http://localhost:3000/api/analisis/1
```

```json
{
  "success": true,
  "data": {
    "id": 7,
    "tipo": "profundo",
    "estado": "completo",
    "generado_en": "2026-03-15T10:30:00.000Z",
    "capas": [
      {
        "nombre_capa": "narrativa",
        "contenido": "La película explora el duelo a través de...",
        "orden": 1,
        "modo": "profundo"
      }
    ]
  }
}
```
