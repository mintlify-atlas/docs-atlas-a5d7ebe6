<Note>
  Magic Film is currently at **version 0.1 (interactive mockup)**. The backend, API integrations, and database connection are under active development. Some steps below describe functionality that is scaffolded but not yet fully operational.
</Note>

## Prerequisites

Before you begin, make sure you have the following:

- **Node.js 18+** — [nodejs.org](https://nodejs.org)
- **MySQL 8+** — [mysql.com](https://mysql.com)
- **TMDB API key** — Create a free account at [themoviedb.org/settings/api](https://www.themoviedb.org/settings/api)
- **Google Gemini API key** — Generate one at [aistudio.google.com](https://aistudio.google.com)

## Setup steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Michael5006/magic-film.git
    cd magic-film
    ```
  </Step>

  <Step title="Install dependencies">
    From the project root, install the backend Node.js packages:

    ```bash
    npm install
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example file and fill in your values:

    ```bash
    cp .env.example .env
    ```

    Open `.env` and replace the placeholder values:

    ```bash .env
    # ── SERVER ────────────────────────────────────
    PORT=3000
    NODE_ENV=development

    # ── DATABASE ──────────────────────────────────
    DB_HOST=localhost
    DB_PORT=3306
    DB_NAME=magic_film
    DB_USER=root
    DB_PASSWORD=password_aqui

    # ── EXTERNAL APIs ─────────────────────────────
    # Get your key at: https://www.themoviedb.org/settings/api
    TMDB_API_KEY=clave_tmdb_aqui
    TMDB_BASE_URL=https://api.themoviedb.org/3

    # Get your key at: https://aistudio.google.com
    GEMINI_API_KEY=clave_gemini_aqui
    GEMINI_MODEL=gemini-1.5-flash

    # ── SECURITY ──────────────────────────────────
    JWT_SECRET=your_strong_secret_here
    JWT_EXPIRES_IN=7d
    ```

    Replace `password_aqui`, `clave_tmdb_aqui`, `clave_gemini_aqui`, and `your_strong_secret_here` with your actual values.
  </Step>

  <Step title="Create the database">
    Connect to your MySQL instance and create the database:

    ```sql
    CREATE DATABASE IF NOT EXISTS magic_filmv01
      CHARACTER SET utf8mb4
      COLLATE utf8mb4_unicode_ci;
    ```

    Or use the MySQL CLI:

    ```bash
    mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS magic_filmv01 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
    ```
  </Step>

  <Step title="Run the database schema">
    Apply the schema to create all tables, indexes, and seed data:

    ```bash
    mysql -u root -p magic_filmv01 < database/schema.sql
    ```

    This creates all required tables (`usuarios`, `peliculas`, `analisis`, `capas_analisis`, `favoritos`, and more) and inserts the initial genre list and default admin user.
  </Step>

  <Step title="Start the backend server">
    ```bash
    npm start
    ```

    Or run directly with Node:

    ```bash
    node backend/server.js
    ```

    The server will start on `http://localhost:3000` (or the `PORT` value from your `.env`).
  </Step>

  <Step title="Open the frontend">
    The frontend is a static HTML/CSS/JS application. Open the entry point directly in your browser:

    ```bash
    open frontend/pages/index.html
    ```

    On Linux or Windows, navigate to `frontend/pages/index.html` in your file manager or open it from your browser with `File > Open`.

    Alternatively, if the backend serves static files, visit `http://localhost:3000` directly.
  </Step>
</Steps>
